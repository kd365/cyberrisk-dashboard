### Workflow Connections

| Workflow | Trigger | AWS Services Used |
|----------|---------|-------------------|
| `ci-backend.yml` | Push/PR to `backend/**` | None (runs tests only) |
| `ci-frontend.yml` | Push/PR to `frontend/**` | None (runs tests only) |
| `deploy-backend.yml` | After `ci-backend` succeeds on `main` | ECR, SSM, EC2 |
| `deploy-frontend.yml` | After `ci-frontend` succeeds on `main` | S3, CloudFront |
| `db-migration.yml` | Manual trigger (workflow_dispatch) | RDS, SSM, EC2 |
| `health-check.yml` | Scheduled (cron) | EC2 (via HTTP) |

### RDS Table: document_chunks

```sql
CREATE TABLE document_chunks (
    id SERIAL PRIMARY KEY,
    document_id VARCHAR(255),      -- S3 key or unique ID
    ticker VARCHAR(10),            -- Company ticker
    document_type VARCHAR(50),     -- '10-K', '10-Q', 'transcript'
    chunk_index INTEGER,           -- Position in document
    content TEXT,                  -- Chunk text
    embedding vector(1536),        -- Titan embedding
    metadata JSONB,                -- Additional metadata
    created_at TIMESTAMP
);

-- Vector similarity search index
CREATE INDEX ON document_chunks
USING ivfflat (embedding vector_cosine_ops);
```

## 10. GitHub Secrets Required

For the architecture to work, these secrets must be configured in GitHub:

| Secret | Purpose | Used By |
|--------|---------|---------|
| `AWS_ACCESS_KEY_ID` | AWS authentication | All workflows |
| `AWS_SECRET_ACCESS_KEY` | AWS authentication | All workflows |
| `EC2_INSTANCE_ID` | Target for SSM commands | deploy-backend, db-migration |
| `DB_HOST` | RDS endpoint | db-migration |
| `DB_PORT` | RDS port (5432) | db-migration |
| `DB_NAME` | Database name | db-migration |
| `DB_USER` | Database username | db-migration |
| `DB_PASSWORD` | Database password | db-migration |
| `BACKEND_URL` | (Optional) Direct API URL | db-migration |

## 11. Environment Variables on EC2

The EC2 instance needs these environment variables (stored in `/opt/cyberrisk/.env`):

```bash
# AWS
AWS_REGION=us-west-2

# Database (RDS)
DB_HOST=cyberrisk-db.xxxxx.us-west-2.rds.amazonaws.com
DB_PORT=5432
DB_NAME=cyberrisk
DB_USER=postgres
DB_PASSWORD=xxxxx

# Neo4j
NEO4J_URI=bolt://neo4j-instance:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=xxxxx

# S3
S3_BUCKET=cyberrisk-documents

# Cognito (Authentication)
COGNITO_USER_POOL_ID=us-west-2_xxxxx
COGNITO_CLIENT_ID=xxxxx

# External APIs
CORESIGNAL_API_KEY=xxxxx

# Feature Flags
USE_LANGCHAIN=true

# Mem0 (if using)
MEM0_API_KEY=xxxxx
```
